## Competency Areas Covered

**Competency 1 — Algorithmic Thinking and Programming**
- Algorithm fundamentals (definition, steps, termination, unambiguity)
- IPO model (Input, Process, Output)
- Variables and data types in Python
- Conditional logic (`if`/`elif`/`else`)
- For loops and while loops
- Functions (`def`, `return`, arguments)
- String operations and indexing
- List operations (`append`, `sort`, `min`, `max`, `sum`)
- Algorithm design patterns (linear scan, running max/min, Fibonacci, Sieve)
- Search algorithms (linear vs. binary)
- Python syntax basics: `print()`, comments, indentation, operator precedence
- Programming language concepts: interpreter, high-level vs. low-level, Python benefits

**Competency 2 — Troubleshooting and Debugging**
- Error classification (syntax, runtime, logic)
- Reading Python tracebacks
- Hypothesis testing and targeted `print()` debugging
- IDE features: linting, static analysis
- Effective AI debugging prompts
- Validation strategies after applying fixes

**Competency 3 — Relational Databases and SQL**
- Relational database fundamentals
- Primary and foreign keys
- SQL `SELECT`, `WHERE`, `INSERT`, `UPDATE`
- Database relationships: one-to-one, one-to-many, many-to-many
- Normalization: 1NF atomicity rule
- Database security: SQL injection

## How to Use This Exam

### Sections C1–C3 (Multiple choice)
1. Click any section button at the top of the exam page
2. Read each question carefully — many include Python code you must trace
3. Click your answer
4. The correct answer, full explanation, and instructor note appear immediately
5. Use the tab row to jump between questions or track what you have answered
6. Monitor your competency breakdown bars at the bottom

### Coding Assessment Section
This section requires **PyCharm**, **VS Code**, or any Python IDE installed on your computer.

1. Read the challenge description carefully
2. Write your Python program in your IDE from scratch
3. Run it and verify your output matches the expected format
4. Paste your code into the **Step 1** box in the browser
5. Paste your program's actual terminal output into the **Step 2** box
6. Click **Check my output** — the exam compares your output line-by-line
7. Click **Mark correct** or **Mark incorrect** based on the result
8. If stuck, use **Show hint** first, then **Show solution** as a last resort

> **Treat the coding section as the real exam.** Writing complete programs from scratch — without looking at the solution — is exactly what the official assessment requires.
